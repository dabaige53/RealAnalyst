# Summary Structured Example

## 适用场景
- 中等复杂度的经营复盘
- 需要“先总览、再分维度、再总结”的阅读体验
- 读者既需要快速把握结论，也需要继续往下看细节

## 阅读目标
- 开头给出总判断
- 中间分维度展开证据
- 结尾把分散结论重新收束

## 开头写法
- 开头应该像“摘要 + 阅读导航”
- 要告诉读者：整体结论是什么，后面会从哪几个维度展开

```markdown
本期整体表现较上期走弱，但不同维度分化明显：直营渠道保持增长，代理渠道回落，重点压力集中在华东市场和中段产品线。以下将分别从渠道、区域和产品三个维度展开，说明本次变化主要来自哪里，以及哪些变化值得继续跟踪。
```

## 证据块写法
- 每个分维度章节都要有自己的证据块
- 证据块不追求全，而要能支撑“这个维度的主要判断”

```markdown
### 渠道表现

| 渠道 | 本期订单量 | 上期订单量 | 变化 |
| --- | --- | --- | --- |
| 直营网 | 12.8万 | 12.1万 | +5.8% |
| 代理渠道 | 9.4万 | 11.0万 | -14.5% |

判断：整体下滑主要集中在代理渠道。
解释：直营保持增长，但代理渠道明显回落，说明总量压力并非全渠道普遍发生，而是渠道结构在变化。
```

## 章节组织建议
- `总体概况`
- `渠道分析`
- `区域分析`
- `产品/产品分析`
- `综合结论`

## 正文与附件边界
- 正文：每个维度 1 个核心证据块 + 1 段解释
- 附件：该维度的完整长表、交叉明细、补充排名

## 不要这样写
- 每个维度只写一句结论，没有数字
- 把“详细分析”写成附件目录
- 用空泛句子代替趋势和差异说明
- 结尾不收束，导致读者看完仍然不知道总问题是什么
