# Ranking Report Example

## 开头写法
- 开头先说清楚“谁最好、谁最差、差距有多大”。
- 重点是帮助读者快速抓住头尾部，不是穷举完整名单。

## 证据块写法
```markdown
### Top / Bottom

| 排名 | 对象 | 订单量 | 变化 |
| --- | --- | --- | --- |
| 1 | 代理人A | 2.4万 | +8.2% |
| 2 | 代理人B | 2.1万 | +4.5% |

| 排名 | 对象 | 订单量 | 变化 |
| --- | --- | --- | --- |
| 倒1 | 代理人Y | 0.3万 | -22.4% |
| 倒2 | 代理人Z | 0.4万 | -18.1% |

判断：头部仍然稳定，但尾部恶化速度更快，需要优先关注倒数对象。
解释：头部波动较小，真正拖累整体的是尾部对象集中下滑。
```

## 不要这样写
- 把完整 Top50 全塞进正文。
- 只列排名，不解释头尾部意味着什么。
- 只给 Top，不给 Bottom。
