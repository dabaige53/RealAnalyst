# Goal Contract

## 字段定义

### `artifact`

| 值 | 含义 | 使用场景 |
|-----|------|---------|
| `csv` | 独立 CSV 文件 | 排名汇总、交叉分析、需要完整数据的场景 |
| `markdown` | 嵌入报告的 Markdown 表格 | 小数据量、趋势表 |
| `json` | JSON 格式（少用） | 结构化数据交换 |

### `filename`

使用中文命名：

- 汇总表：`汇总_{维度}_{time_range}.csv`
- 交叉表：`交叉_{维度1}×{维度2}_{time_range}.csv`

### `params`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `top_n` | int | 20 | 排名展示数量（仅用于报告展示，CSV 导出全量） |
| `threshold` | float | - | 筛选阈值 |
| `aggregation` | string | sum | 聚合方式（sum/avg/count） |

示例：

```yaml
params: { top_n: 20, aggregation: sum }
```

## 示例

### 维度排名分析

```markdown
### 维度目标
- [ ] goal-dim-代理人: 按代理人维度分析结构分布与 Top/Bottom
  - artifact: csv
  - filename: 汇总_代理人_2025Q4.csv
  - params: { top_n: 20 }
```

### 交叉分析

```markdown
### 交叉目标
- [ ] goal-cross-1: 月份×产品交叉分析
  - artifact: csv
  - filename: 交叉_月份×产品_2025Q4.csv
```

## 使用规则

### 必须添加契约字段的场景

| 目标类型 | 是否必须 | 原因 |
|---------|---------|------|
| 维度目标 | ✅ 必须 | 排名类输出，需明确文件名和 top_n |
| 交叉目标 | ✅ 必须 | 交叉表输出，需明确文件名 |
| 假设验证目标 | ⚠️ 视情况 | 若涉及数据导出，需添加 |
| 固定目标 | ❌ 可选 | 通常为概览，嵌入报告即可 |
| 总结目标 | ❌ 可选 | 文字总结，无需独立文件 |

### 契约字段与输出决策的关系

**优先级**：plan 契约 > 行数阈值

| 场景 | plan 契约 | 行数 | 输出决策 |
|------|----------|------|---------|
| 排名汇总 | `artifact: csv` | 任意 | ✅ 导出 CSV（全量） |
| 排名汇总 | 无契约 | ≤ 50 | ❌ 嵌入报告 |
| 排名汇总 | 无契约 | > 50 | ✅ 导出 CSV |
| 交叉分析 | `artifact: csv` | 任意 | ✅ 导出 CSV（全量） |
| 交叉分析 | 无契约 | ≤ 20 | ❌ 嵌入报告 |
| 交叉分析 | 无契约 | > 20 | ✅ 导出 CSV |

关键规则：

- 若 plan 明确要求 `artifact: csv`，无论行数都导出
- 排名类 CSV 导出全量数据，Top N 仅用于报告展示
- `params.top_n` 仅影响报告展示层，不影响 CSV 导出
