# Tableau 筛选器使用指南

> ⚠️ **重要声明**
>
> 本文档仅为**格式示例**，不是筛选 key 的权威来源。
>
> **权威来源**：`{baseDir}/runtime/registry.db` 的运行时查询结果
>
> 使用前必须通过 `query_registry.py --filter <source_key>` 获取当前数据源支持的 key。

## 筛选器与参数传递方式

```bash
python3 {baseDir}/skills/data-export/scripts/tableau/export_source.py --source-id <source_id> \
  --vf "筛选器字段=值" \
  --vp "参数名=值"
```

### --vf vs --vp 区别

| 类型 | 参数 | 用途 | 示例 |
|------|------|------|------|
| **Filter** | `--vf` | 离散字段筛选（在 `filters:` 中定义） | `--vf "代理_区域=上海区域"` |
| **Parameter** | `--vp` | 日期范围等参数（在 `parameters:` 中定义） | `--vp "出票日期_开始=2025-10-01"` |

> 查看 `query_registry.py --filter <source_key>` 获取当前 source 的正确字段 token。

## 筛选器值格式规范

### 日期类筛选器

| 字段名 | 格式 | 单值示例 | 多值示例 |
|--------|------|----------|----------|
| `出票日期年月` | YYYYMM | `202511` | `202510,202511,202512` |

**季度转换**:
- Q1: `202501,202502,202503`
- Q2: `202504,202505,202506`
- Q3: `202507,202508,202509`
- Q4: `202510,202511,202512`

> 枚举型筛选字段支持多值时，可在单次 `--vf` 中用逗号传多个值，例如：`--vf "出票日期年月=202501,202502,202503"`。

## Fallback: 本地筛选

当 Tableau 筛选器不可用或格式未知时，使用本地筛选策略：

### 步骤

1. **下载全量数据**（不带筛选器）
```bash
python3 {baseDir}/skills/data-export/scripts/tableau/export_source.py --source-id <source_id>
```

2. **本地筛选** — Agent 直接读取 CSV 并筛选
```python
# Agent 伪代码示例
import pandas as pd
df = pd.read_csv("jobs/{SESSION_ID}/data/交叉_<source_key>.csv")
filtered = df[
    (df["代理-区域"] == "上海区域") &
    (df["出票日期年月"].isin(["202510", "202511", "202512"]))
]
```

### 何时使用 Fallback

| 情况 | 处理方式 |
|------|----------|
| 筛选器字段名未知 | 先下载采样，查看 CSV 列名 |
| 筛选器值格式未知 | 先下载采样，查看该列的 unique values |
| 筛选器不生效 | 下载全量，本地筛选 |
| 数据量过大（>100万行） | 尝试用筛选器减少数据量 |

## 新增筛选器验证流程

当遇到未记录的筛选器时：

1. **获取 CSV 采样**
```bash
python3 {baseDir}/skills/data-export/scripts/tableau/export_source.py --source-id <source_id>
```

2. **检查列名和值**
```bash
head -1 jobs/{SESSION_ID}/data/交叉_<source_key>.csv  # 查看列名
cut -d',' -f3 jobs/{SESSION_ID}/data/交叉_<source_key>.csv | sort -u | head -20  # 查看第3列的值
```

3. **测试筛选器**
```bash
python3 {baseDir}/skills/data-export/scripts/tableau/export_source.py --source-id <source_id> \
  --vf "字段名=测试值"
```

4. **验证筛选效果**
```bash
# 对比行数
wc -l jobs/{SESSION_ID}/data/交叉_<source_key>.csv
```

5. **更新运行时配置** — 将验证通过的筛选器同步回 registry/spec 的正式维护链路，确保后续 `query_registry.py` 可直接返回。
